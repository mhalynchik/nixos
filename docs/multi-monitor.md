## Определение мониторов

### Получение информации о мониторах

```bash
# Список всех подключенных мониторов
hyprctl monitors all

# Подробная информация с описаниями
hyprctl monitors all | grep -E "Monitor|description|resolution|position"
```

Пример вывода:
```
Monitor DP-4 (ID 0):
    2560x1440@143.99800 at 0x0
    description: Dell Inc. DELL S2722DGM (DP-4)
Monitor HDMI-A-1 (ID 1):
    1920x1080@60.00000 at 2560x0
    description: Samsung Electric Company (HDMI-A-1)
```

### Формат конфигурации монитора

```
name,resolution@refresh,position,scale
```

| Параметр | Описание | Примеры |
|----------|----------|---------|
| `name` | Имя монитора из `hyprctl monitors` | `DP-4`, `HDMI-A-1`, `eDP-1` |
| `resolution` | Разрешение экрана | `2560x1440`, `1920x1080` |
| `refresh` | Частота обновления (Гц) | `144`, `60`, `75` |
| `position` | Позиция X,Y в пикселях | `0x0`, `2560x0`, `-1920x0` |
| `scale` | Масштаб (1 = 100%) | `1`, `1.5`, `2` |

## Настройка в vars.nix

### Основной монитор

```nix
# Главный монитор (всегда должен быть указан)
monitor = "DP-4,2560x1440@144,0x0,1";
```

### Дополнительные мониторы

```nix
additionalMonitors = [
  # Второй монитор справа от основного
  "HDMI-A-1,1920x1080@60,2560x0,1"
  
  # Третий монитор слева от основного
  # "DP-2,1920x1080@60,-1920x0,1"
];
```

### Расположение мониторов

```
Позиция рассчитывается от левого верхнего угла (0,0)

        ┌──────────────┐
        │   -1920x0    │  <- Монитор слева
        │  1920x1080   │
┌───────┼──────────────┼───────────┐
│ 0x0   │              │  2560x0   │  <- Монитор справа
│2560x1440             │ 1920x1080 │
│ (main)│              │           │
└───────┴──────────────┴───────────┘
```

**Примеры позиций:**
- `0x0` — основной монитор (точка отсчёта)
- `2560x0` — справа от 2560px монитора
- `-1920x0` — слева от основного
- `0x1440` — снизу от 1440px монитора
- `0x-1080` — сверху от основного

## Привязка workspaces к мониторам

По умолчанию workspaces могут перемещаться между мониторами. Чтобы закрепить их:

```nix
workspaceMonitorBindings = [
  # Формат: "номер_workspace, имя_монитора"
  
  # Основной монитор (DP-4)
  "1, DP-4"
  "2, DP-4"
  "3, DP-4"
  "4, DP-4"
  "5, DP-4"
  
  # Второй монитор (HDMI-A-1)
  "6, HDMI-A-1"
  "7, HDMI-A-1"
  "8, HDMI-A-1"
  "9, HDMI-A-1"
  "10, HDMI-A-1"
];
```

> **Совет:** Workspaces, не указанные в списке, будут "плавающими" и могут появляться на любом мониторе.

## Горячие клавиши

### Переключение фокуса между мониторами

| Клавиши | Действие |
|---------|----------|
| `Super + Ctrl + ←` | Фокус на левый монитор |
| `Super + Ctrl + →` | Фокус на правый монитор |
| `Super + Ctrl + ↑` | Фокус на верхний монитор |
| `Super + Ctrl + ↓` | Фокус на нижний монитор |

### Перемещение окон между мониторами

| Клавиши | Действие |
|---------|----------|
| `Super + Ctrl + Shift + ←` | Переместить окно на левый монитор |
| `Super + Ctrl + Shift + →` | Переместить окно на правый монитор |
| `Super + Ctrl + Shift + ↑` | Переместить окно на верхний монитор |
| `Super + Ctrl + Shift + ↓` | Переместить окно на нижний монитор |

### Обмен workspaces между мониторами

| Клавиши | Действие |
|---------|----------|
| `Super + Ctrl + Alt + ←` | Поменять workspace с левым монитором |
| `Super + Ctrl + Alt + →` | Поменять workspace с правым монитором |

## Поведение компонентов

### Waybar
- Отображается на **каждом** мониторе
- Показывает только workspaces текущего монитора
- Независимая панель на каждом экране

### Уведомления (Dunst/SwayNC)
- Следуют за **курсором мыши**
- Появляются на мониторе, где находится курсор

### AGS Popups
- Открываются на **текущем активном** мониторе
- Автоматически перемещаются при смене фокуса

### nwg-dock
- Следует за **активным монитором** (флаг `-f`)
- Автоматически перемещается при переключении

### Hyprlock (экран блокировки)
- Отображается на **всех** мониторах одновременно
- Ввод пароля доступен на любом экране

### Обои (swww/mpvpaper)
- Устанавливаются на **все** мониторы
- Используется символ `*` для применения ко всем

## Примеры конфигураций

### Рабочее место: основной + правый монитор

```nix
{
  # Основной монитор 2K 144Hz
  monitor = "DP-4,2560x1440@144,0x0,1";
  
  # Дополнительный Full HD справа
  additionalMonitors = [
    "HDMI-A-1,1920x1080@60,2560x0,1"
  ];
  
  # Workspaces: 1-5 на основном, 6-10 на дополнительном
  workspaceMonitorBindings = [
    "1, DP-4"  "2, DP-4"  "3, DP-4"  "4, DP-4"  "5, DP-4"
    "6, HDMI-A-1"  "7, HDMI-A-1"  "8, HDMI-A-1"  "9, HDMI-A-1"  "10, HDMI-A-1"
  ];
}
```

### Ноутбук + внешний монитор сверху

```nix
{
  # Встроенный экран ноутбука
  monitor = "eDP-1,1920x1080@60,0x1440,1";
  
  # Внешний монитор сверху
  additionalMonitors = [
    "HDMI-A-1,2560x1440@60,0x0,1"
  ];
  
  workspaceMonitorBindings = [
    # Внешний — для основной работы
    "1, HDMI-A-1"  "2, HDMI-A-1"  "3, HDMI-A-1"
    # Встроенный — для мессенджеров и музыки
    "8, eDP-1"  "9, eDP-1"  "10, eDP-1"
  ];
}
```

### Три монитора (центр + лево + право)

```nix
{
  monitor = "DP-1,2560x1440@144,1920x0,1";  # Центр
  
  additionalMonitors = [
    "DP-2,1920x1080@60,0x0,1"       # Слева
    "HDMI-A-1,1920x1080@60,4480x0,1" # Справа (2560+1920=4480)
  ];
  
  workspaceMonitorBindings = [
    "1, DP-2"  "2, DP-2"  "3, DP-2"      # Левый
    "4, DP-1"  "5, DP-1"  "6, DP-1"      # Центральный
    "7, HDMI-A-1"  "8, HDMI-A-1"  "9, HDMI-A-1"  # Правый
  ];
}
```

### Автоопределение (для динамических подключений)

```nix
{
  # Автоматическое определение разрешения и позиции
  monitor = ",preferred,auto,1";
  
  additionalMonitors = [
    ",preferred,auto,1"  # Все мониторы с авто-настройкой
  ];
  
  # Без привязки — workspaces свободно перемещаются
  workspaceMonitorBindings = [];
}
```

## Решение проблем

### Монитор не определяется

1. Проверьте подключение кабеля
2. Выполните `hyprctl monitors all`
3. Проверьте логи: `journalctl -b | grep -i monitor`

### Неправильная позиция монитора

1. Уточните разрешение: `hyprctl monitors | grep resolution`
2. Пересчитайте позицию:
   - Справа: `X = ширина_левого_монитора`
   - Слева: `X = -ширина_этого_монитора`
   - Снизу: `Y = высота_верхнего_монитора`

### Workspaces "прыгают" между мониторами

Добавьте привязку в `workspaceMonitorBindings`.

### Dock не перемещается между мониторами

Убедитесь, что nwg-dock запущен с флагом `-f`:
```nix
"nwg-dock-hyprland -r -f -i 48 -mb 8 -ml 8 -mr 8"
```

### Масштабирование на HiDPI мониторе

```nix
# Для 4K монитора
monitor = "DP-4,3840x2160@60,0x0,1.5";  # scale = 1.5 (150%)

# Или для точного масштаба
monitor = "DP-4,3840x2160@60,0x0,2";    # scale = 2 (200%)
```

### Разные частоты обновления

Hyprland корректно работает с мониторами разной частоты:
```nix
monitor = "DP-4,2560x1440@144,0x0,1";      # 144 Hz
additionalMonitors = [
  "HDMI-A-1,1920x1080@60,2560x0,1"         # 60 Hz — OK
];
```

## Применение изменений

После редактирования `vars.nix`:

```bash
# Пересобрать и применить конфигурацию
sudo nixos-rebuild switch

# Или для тестирования без сохранения
sudo nixos-rebuild test
```

> **Важно:** Некоторые изменения (особенно Waybar и AGS) требуют перезапуска Hyprland:
> - Выйти: `Super + M`
> - Или перелогиниться

## Полезные команды

```bash
# Информация о мониторах
hyprctl monitors all

# Список окон и их мониторов
hyprctl clients | grep -E "class:|monitor:"

# Перезапустить Waybar
pkill waybar && waybar &

# Перезапустить nwg-dock
pkill nwg-dock && nwg-dock-hyprland -r -f -i 48 -mb 8 -ml 8 -mr 8 &

# Сменить обои на всех мониторах
swww img ~/wallpapers/image.jpg
```
